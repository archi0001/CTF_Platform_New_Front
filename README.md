# CTFGuap — Платформа для соревнований по информационной безопасности

Веб-платформа, предназначенная для проведения и участия в соревнованиях формата Capture The Flag (CTF). Проект позволяет организаторам управлять заданиями и отслеживать успехи участников, а командам — решать задачи, зарабатывать очки и соревноваться в турнирной таблице.

## Функциональные возможности и разделы

Платформа состоит из нескольких ключевых модулей, каждый из которых отвечает за отдельную часть игрового процесса и администрирования.

### Главная страница и новости

Раздел для публикации важных объявлений, расписания предстоящих турниров и подведения итогов прошедших соревнований. Пользователи всегда остаются в курсе последних событий площадки.

**Возможности:**

- Просмотр ленты новостей.
- Пагинация или бесконечная прокрутка для старых записей.

Основной раздел платформы, где участники выбирают доступные турниры и приступают к решению тасков (заданий).

**Возможности:**

- Отображение списка активных и прошедших соревнований.
- Категоризация заданий (Web, Crypto, Pwn, Reverse, Forensics, OSINT и др.).
- Ввод и валидация флагов с начислением баллов.

- Отображение общей статистики: заработанные баллы, рейтинг среди всех участников.

## Технологический стек

Проект разделен на клиентскую и серверную части, каждая из которых написана с использованием современных фреймворков.

**Backend (Серверная часть):**

- **FastAPI** — высокопроизводительный веб-фреймворк для создания API.
- **SQLAlchemy** — ORM для взаимодействия с базой данных.
- **PostgreSQL** — надежная реляционная база данных для хранения информации о пользователях, тасках и логах.
- **bcrypt** — алгоритм хеширования паролей.
- **python-jose** — генерация и валидация JWT токенов для авторизации.
- **Jinja2** — шаблонизатор.

**Frontend (Клиентская часть):**

- **React 19** — библиотека для построения пользовательских интерфейсов.
- **TypeScript** — строгая типизация для предотвращения ошибок на этапе разработки.
- **Vite** — быстрый сборщик модулей.
- **MobX** — инструмент для управления состоянием приложения.
- **React Router** — маршрутизация между страницами без перезагрузки браузера.
- **Axios** — HTTP-клиент для взаимодействия с API.
- **Sass** — препроцессор для удобного написания и структурирования CSS стилей.

## Требования к окружению

Для корректной работы проекта понадобятся следующие инструменты:

- Docker и Docker Compose (для быстрого развертывания).
- Node.js версии 20 и выше (для локального запуска клиентской части).
- Python 3.10 и выше (для локального запуска серверной части).

## Развертывание и запуск

### Запуск через Docker (Рекомендуемый способ)

Для быстрого старта достаточно выполнить несколько команд:

1. Клонируйте репозиторий:

```bash
git clone <repository-url>
cd CTFGuap
```

2. Соберите и запустите контейнеры:

```bash
docker compose up --build
```

Для запуска в фоновом режиме используйте флаг `-d`.

3. Приложение будет доступно по адресам:

- Пользовательский интерфейс: `http://localhost:5173`
- Серверное API: `http://localhost:8040`

Для остановки проекта используйте команду `docker compose down`. Чтобы сбросить данные в базе и пересоздать контейнеры, используйте `docker compose down -v` перед повторным запуском.

### Локальная разработка

**Настройка серверной части (Backend):**

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Для Linux/MacOS (в Windows: venv\Scripts\activate)
pip install -r requirements.txt
uvicorn main:app --reload --host 0.0.0.0 --port 8040
```

Swagger UI с документацией API будет доступен по адресу `http://localhost:8040/docs`.

**Настройка клиентской части (Frontend):**

```bash
cd frontend
npm install
npm run dev
```

**Запуск базы данных PostgreSQL (если не используется готовая):**

```bash
docker run -d \
  -e POSTGRES_USER=myuser \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_DB=mydb \
  -p 5432:5432 \
  postgres:16
```

## API и документация

Взаимодействие между клиентом и сервером происходит по REST API.

Основные эндпоинты:

- `POST /api/login` — авторизация пользователя.
- `POST /api/register` — создание новой учетной записи.
- `GET /api/profile` — данные текущего пользователя.
- `GET /api/events` — список соревнований.
- `GET /api/news` — лента новостей.
- `GET /api/team?team_id={id}` — данные о конкретной команде.
- `GET /api/user?user_id={id}` — данные о конкретном пользователе.

Полная спецификация методов и параметров доступна в интерактивном формате (Swagger UI) по адресу `http://localhost:8040/docs` при запущенном сервере.

## Конфигурация

### Переменные окружения (Backend)

- `DATABASE_URL` — строка подключения к базе данных (по умолчанию: `postgresql://myuser:mypassword@postgres:5432/mydb`).
- `SECRET_KEY` — ключ для подписи JWT токенов.

### Переменные окружения (Frontend)

Создайте или отредактируйте файл `frontend/.env`:

- `VITE_API_URL` — адрес серверного API (по умолчанию: `http://localhost:8040`).

## Текущее состояние проекта

В данный момент обновлен дизайн клиентской части приложения. Серверная часть оставлена в исходном виде.

Обратите внимание, что на некоторых страницах (например, новости, список соревнований и профиль), где требуется получение данных с сервера, запросы к API временно отключены. Вместо них используются статические тестовые данные для демонстрации интерфейса. Для восстановления полной работоспособности необходимо раскомментировать соответствующие вызовы API и убрать статические заглушки. Страница управления командой ранее отсутствовала, поэтому запросы для нее потребуется реализовать. Также требуется настроить обработку нажатий кнопок в панели администратора и на страницах соревнований (включая переадресацию в необходимые разделы платформы CTFd).
